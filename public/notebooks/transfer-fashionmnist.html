1. Transforms y DataLoaders

ResNet18 fue entrenada en ImageNet con imágenes RGB de 224×224 normalizadas con medias/desvíos específicos. FashionMNIST es 28×28 en gris, así que el pipeline de transforms:

1. Resize a 224×224.

2. Grayscale→3 canales (Grayscale(num_output_channels=3)): replica el canal para que entre en la ResNet.

3. ToTensor + Normalize con las estadísticas de ImageNet.

# 🧠 Prompt: "Armá los transforms con resize 224, grayscale a 3 canales y normalización ImageNet."
IMAGENET_MEAN = [0.485, 0.456, 0.406]
IMAGENET_STD  = [0.229, 0.224, 0.225]

tf = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.Grayscale(num_output_channels=3),
    transforms.ToTensor(),
    transforms.Normalize(IMAGENET_MEAN, IMAGENET_STD),
])

train_full = datasets.FashionMNIST(root="./data", train=True,  download=True, transform=tf)
test_full  = datasets.FashionMNIST(root="./data", train=False, download=True, transform=tf)
CLASSES = train_full.classes
print("Clases:", CLASSES)
# 🧠 Prompt: "Tomá un subconjunto balanceado para demo rápida y armá los DataLoaders de train y test."
# --- Config reducida para demo en Colab; subir estos números para entrenamiento completo ---
N_TRAIN = 3000
N_TEST  = 1000
BATCH   = 64

g = torch.Generator().manual_seed(SEED)
train_idx = torch.randperm(len(train_full), generator=g)[:N_TRAIN]
test_idx  = torch.randperm(len(test_full),  generator=g)[:N_TEST]

train_ds = Subset(train_full, train_idx.tolist())
test_ds  = Subset(test_full,  test_idx.tolist())

train_loader = DataLoader(train_ds, batch_size=BATCH, shuffle=True,  num_workers=2)
test_loader  = DataLoader(test_ds,  batch_size=BATCH, shuffle=False, num_workers=2)
print(f"Train: {len(train_ds)} | Test: {len(test_ds)} | batches train: {len(train_loader)}")

3. Modelo: ResNet18 preentrenada (Transfer Learning)

Estrategia de feature extraction + fine-tuning ligero:

  • Cargamos ResNet18 con pesos de ImageNet.
  • Congelamos todo el backbone (no se actualizan sus pesos al principio).
  • Reemplazamos la capa final fc (1000 clases de ImageNet) por una Linear de 10 salidas (nuestras prendas). Esa capa nueva sí se entrena.
  • Opcionalmente descongelamos el último bloque para fine-tuning fino.
# 🧠 Prompt: "Cargá resnet18 con pesos ImageNet, congelá parámetros y reemplazá fc por Linear de 10 clases."
def build_model(n_classes=10, unfreeze_layer4=True):
    weights = models.ResNet18_Weights.IMAGENET1K_V1
    model = models.resnet18(weights=weights)
    for p in model.parameters():
        p.requires_grad = False           # congelar backbone
    if unfreeze_layer4:
        for p in model.layer4.parameters():
            p.requires_grad = True        # fine-tuning del último bloque
    in_feat = model.fc.in_features
    model.fc = nn.Linear(in_feat, n_classes)   # cabeza nueva (entrenable)
    return model

model = build_model().to(device)
n_train = sum(p.numel() for p in model.parameters() if p.requires_grad)
n_total = sum(p.numel() for p in model.parameters())
print(f"Parámetros entrenables: {n_train:,} de {n_total:,}")

4. Entrenamiento (fine-tuning)

Solo optimizamos los parámetros con requires_grad=True (la cabeza nueva + layer4). Usamos Adam y CrossEntropyLoss.

# 🧠 Prompt: "Definí evaluate y train que registren loss y accuracy de train y test por época."
@torch.no_grad()
def evaluate(model, loader, criterion):
    model.eval()
    loss_sum, correct, n = 0.0, 0, 0
    for xb, yb in loader:
        xb, yb = xb.to(device), yb.to(device)
        out = model(xb)
        loss_sum += criterion(out, yb).item() * xb.size(0)
        correct += (out.argmax(1) == yb).sum().item()
        n += xb.size(0)
    return loss_sum / n, correct / n

def train(model, train_loader, test_loader, epochs=4, lr=1e-3):
    criterion = nn.CrossEntropyLoss()
    params = [p for p in model.parameters() if p.requires_grad]
    optimizer = torch.optim.Adam(params, lr=lr)
    hist = {"tr_loss": [], "te_loss": [], "tr_acc": [], "te_acc": []}
    for ep in range(epochs):
        model.train()
        for xb, yb in train_loader:
            xb, yb = xb.to(device), yb.to(device)
            optimizer.zero_grad()
            loss = criterion(model(xb), yb)
            loss.backward(); optimizer.step()
        trl, tra = evaluate(model, train_loader, criterion)
        tel, tea = evaluate(model, test_loader, criterion)
        for k, v in zip(hist, [trl, tel, tra, tea]): hist[k].append(v)
        print(f"Época {ep+1}/{epochs} | train loss {trl:.3f} acc {tra:.3f} | test loss {tel:.3f} acc {tea:.3f}")
    return hist
# 🧠 Prompt: "Entrená 4 épocas (config demo) y graficá las curvas de loss y accuracy."
EPOCHS = 4   # subir a 10-15 en GPU completa
hist = train(model, train_loader, test_loader, epochs=EPOCHS, lr=1e-3)

fig, ax = plt.subplots(1, 2, figsize=(14, 5))
ax[0].plot(hist["tr_loss"], "o-", label="train"); ax[0].plot(hist["te_loss"], "s-", label="test")
ax[0].set_title("Loss"); ax[0].set_xlabel("época"); ax[0].legend()
ax[1].plot(hist["tr_acc"], "o-", label="train"); ax[1].plot(hist["te_acc"], "s-", label="test")
ax[1].set_title("Accuracy"); ax[1].set_xlabel("época"); ax[1].legend()
plt.tight_layout(); plt.show()

5. Evaluación final

# 🧠 Prompt: "Generá predicciones sobre todo el test y mostrá classification_report y matriz de confusión."
model.eval()
y_true, y_pred = [], []
with torch.no_grad():
    for xb, yb in test_loader:
        out = model(xb.to(device))
        y_pred.extend(out.argmax(1).cpu().tolist())
        y_true.extend(yb.tolist())

print(classification_report(y_true, y_pred, target_names=CLASSES))

cm = confusion_matrix(y_true, y_pred)
plt.figure(figsize=(10, 8))
sns.heatmap(cm, annot=True, fmt="d", cmap="Blues",
            xticklabels=CLASSES, yticklabels=CLASSES)
plt.title("Matriz de confusión — ResNet18 fine-tuned")
plt.ylabel("real"); plt.xlabel("predicho")
plt.xticks(rotation=45, ha="right"); plt.tight_layout(); plt.show()

Lectura de resultados. Con apenas 4 épocas y un subconjunto chico, el transfer learning ya alcanza una accuracy alta (~85-90%). En la matriz de confusión las confusiones típicas aparecen entre clases visualmente parecidas: *Shirt* vs *T-shirt/Coat/Pullover* (todas prendas superiores) y *Sneaker* vs *Ankle boot*. Esto muestra que el modelo aprende forma/textura de la prenda, no su semántica exacta — lo esperable de features de ImageNet adaptadas.

6. Conclusiones

  • Qué construí: un clasificador de prendas con transfer learning (ResNet18 ImageNet) sobre FashionMNIST, con DataLoaders, fine-tuning del último bloque + cabeza nueva, curvas y matriz de confusión.
  • Detalle técnico clave: ResNet espera 3 canales RGB de 224×224; FashionMNIST es 1 canal de 28×28. Lo resolví con Grayscale(num_output_channels=3) + Resize(224) y la normalización de ImageNet (no la del dataset propio), porque los pesos preentrenados asumen esa distribución de entrada.
  • Por qué transfer learning: congelando el backbone entreno muy pocos parámetros y converjo en pocas épocas, frente a entrenar una CNN desde cero que necesitaría muchos más datos y tiempo.
  • Dificultad superada: al principio pasaba el tensor de 1 canal directo a la ResNet → error de shape [N,1,...] vs [N,3,...]. La solución fue el transform Grayscale(3).
  • Duda que queda: cuánto mejora descongelar todo el backbone (full fine-tuning) vs solo layer4, y a partir de qué tamaño de dataset conviene.
