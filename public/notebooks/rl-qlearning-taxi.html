2. Agente Q-learning tabular

Implemento el agente al estilo del tutorial de Blackjack: una tabla Q (defaultdict), política ε-greedy con decaimiento de ε, y la actualización por la ecuación de Bellman:

$$Q(s,a) \leftarrow Q(s,a) + \alpha \big[ r + \gamma \max_{a'} Q(s',a') - Q(s,a) \big]$$

# 🧠 Prompt: "Definí la clase QLearningAgent con get_action epsilon-greedy, update Bellman y decay_epsilon."
class QLearningAgent:
    def __init__(self, n_actions, lr=0.1, gamma=0.99,
                 eps_start=1.0, eps_min=0.05, eps_decay=0.0005):
        self.n_actions = n_actions
        self.lr = lr
        self.gamma = gamma
        self.epsilon = eps_start
        self.eps_min = eps_min
        self.eps_decay = eps_decay
        self.q = defaultdict(lambda: np.zeros(n_actions))

    def get_action(self, state, greedy=False):
        if (not greedy) and np.random.random() < self.epsilon:
            return np.random.randint(self.n_actions)   # exploración
        return int(np.argmax(self.q[state]))           # explotación

    def update(self, s, a, r, s_next, done):
        target = r + (0 if done else self.gamma * np.max(self.q[s_next]))
        self.q[s][a] += self.lr * (target - self.q[s][a])

    def decay_epsilon(self):
        self.epsilon = max(self.eps_min, self.epsilon - self.eps_decay)

3. Entrenamiento

Loop estándar de RL: por cada episodio el agente actúa, observa (s', r, done), actualiza su Q y decae ε. Registramos la recompensa total y si el episodio fue exitoso (terminó por entrega correcta, no por truncamiento).

# 🧠 Prompt: "Entrená el agente N_EPISODIOS guardando recompensa por episodio y flag de éxito."
N_EPISODIOS = 8000   # suficiente para converger en Taxi; subir si se quiere
agent = QLearningAgent(n_actions, lr=0.1, gamma=0.99,
                       eps_start=1.0, eps_min=0.05, eps_decay=1.0 / (N_EPISODIOS * 0.6))

rewards_hist, exito_hist = [], []
for ep in range(N_EPISODIOS):
    state, _ = env.reset()
    done, total_r = False, 0
    while not done:
        action = agent.get_action(state)
        next_state, reward, terminated, truncated, _ = env.step(action)
        done = terminated or truncated
        agent.update(state, action, reward, next_state, terminated)
        state = next_state
        total_r += reward
    agent.decay_epsilon()
    rewards_hist.append(total_r)
    exito_hist.append(1 if terminated else 0)   # terminated=True -> entrega correcta

    if (ep + 1) % 1000 == 0:
        media = np.mean(rewards_hist[-1000:])
        tasa = np.mean(exito_hist[-1000:])
        print(f"Episodio {ep+1:5d} | recompensa media {media:6.1f} | éxito {tasa:.2%} | eps {agent.epsilon:.3f}")

print("\nEstados visitados en la tabla Q:", len(agent.q))

Lectura de las curvas. Al principio la recompensa es muy negativa (el taxi se mueve al azar y acumula penalizaciones y entregas ilegales). A medida que ε decae y la tabla Q se llena, la recompensa sube y se estabiliza cerca del óptimo (recompensas positivas, pocos pasos) y la tasa de éxito tiende a ~100%. Esa es la firma típica de un Q-learning que convergió.

5. Evaluación de la política

Evaluamos la política aprendida en modo greedy (sin exploración, epsilon=0) sobre varios episodios y medimos recompensa media y pasos promedio.

# 🧠 Prompt: "Evaluá 1000 episodios en modo greedy y reportá recompensa media, pasos medios y tasa de éxito."
def evaluar(agent, env, episodios=1000):
    recompensas, pasos, exitos = [], [], 0
    for _ in range(episodios):
        state, _ = env.reset()
        done, total_r, n = False, 0, 0
        while not done:
            action = agent.get_action(state, greedy=True)
            state, reward, terminated, truncated, _ = env.step(action)
            done = terminated or truncated
            total_r += reward; n += 1
        recompensas.append(total_r); pasos.append(n)
        exitos += int(terminated)
    return np.mean(recompensas), np.mean(pasos), exitos / episodios

r_med, p_med, tasa = evaluar(agent, env, 1000)
print(f"Recompensa media (greedy): {r_med:.2f}")
print(f"Pasos promedio por episodio: {p_med:.2f}")
print(f"Tasa de éxito: {tasa:.2%}")
# 🧠 Prompt: "Renderizá en texto un episodio resuelto por la política aprendida para inspeccionar el comportamiento."
# Render en modo texto (ANSI) de un episodio resuelto con la política aprendida
demo_env = make_taxi(render_mode="ansi")
state, _ = demo_env.reset(seed=123)
done, n = False, 0
print("=== Episodio de demostración (política greedy) ===")
while not done and n < 30:
    action = agent.get_action(state, greedy=True)
    state, reward, terminated, truncated, _ = demo_env.step(action)
    done = terminated or truncated
    n += 1
print(demo_env.render())   # estado final del entorno
print(f"Resuelto en {n} pasos | terminado correctamente: {terminated}")

6. Conclusiones

  • Qué construí: un agente Q-learning tabular completo para Taxi-v3 (tabla Q, ε-greedy con decaimiento, actualización de Bellman), su loop de entrenamiento, las curvas de recompensa y tasa de éxito, y la evaluación de la política aprendida + un episodio de demostración.
  • Adaptación a Colab: Gymnasium se instala con pip install "gymnasium[toy-text]"; el render se hace en modo ansi (texto), ya que Colab no tiene ventana gráfica para human. Para ser robusto a la versión de gymnasium uso una función make_taxi() que prueba Taxi-v3 (la de Colab) y cae a Taxi-v4 (gymnasium reciente), porque la API es idéntica.
  • Aprendizajes: el trade-off exploración/explotación es central — sin decaimiento de ε el agente nunca explota lo aprendido; con un decaimiento demasiado rápido se queda con una política subóptima. El factor de descuento γ alto (0.99) es importante porque la recompensa grande (+20) llega al final del episodio.
  • Dificultad superada: distinguir terminated (entrega correcta = éxito) de truncated (se acabó el tiempo) para medir bien la tasa de éxito y para no descontar el valor futuro tras una terminación real.
  • Duda que queda: cómo escalaría esto a entornos con estados continuos (donde la tabla Q ya no sirve) y haría falta DQN con una red neuronal aproximando Q — el puente natural hacia el deep RL.
